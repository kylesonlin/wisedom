# Project Rules

## Overview

The following project rules will be followed for this project.

## Tone

* Be professional and friendly
* Use a consistent tone throughout the codebase

## Formatting

* Use Markdown for documentation
* Use consistent spacing and indentation

## Code Organization

* Use a modular code structure
* Keep related code together

## Commit Messages

* Use clear and concise commit messages
* Follow the conventional commit message format

## Pull Requests

* Use pull requests to review code changes
* Ensure that code changes are thoroughly reviewed before merging

## Testing

* Write unit tests and integration tests for the code
* Use a testing framework such as Jest or Mocha

## Deployment

* Use a continuous integration and continuous deployment (CI/CD) pipeline to automate deployment
* Ensure that the deployment process is automated and repeatable

## Security

* Follow secure coding practices to prevent common web vulnerabilities such as SQL injection and cross-site scripting (XSS)
* Use secure protocols for data transmission and storage

## Other

* Use Cursor to auto-generate RLS policies
* Use YOLO Mode (Speed Mode) on trusted workflows
